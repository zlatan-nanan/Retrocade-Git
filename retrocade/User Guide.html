01
ABOUT YOUR RETROCADE

Congratulations on your Retrocade arcade cabinet! This machine was designed and built by students from Nanan Academy in Victoria, British Columbia, as part of a STEM education project. Inside is a Raspberry Pi — a small, powerful single-board computer — running RetroPie, a platform that brings classic gaming to life.

The front-end interface you see on screen is called EmulationStation. It organizes your games by system and makes everything controller-navigable — no keyboard or mouse needed.

What's Inside

ComponentDescription
Raspberry PiThe computer at the heart of the cabinet
RetroPieThe operating system and emulator platform
EmulationStationThe visual front-end menu you navigate with the controller
Chromium BrowserRuns HTML/JavaScript games, including student Scratch exports

02
STUDENT-MADE GAMES
🎓

Every game in the Retrocade section was created by a student from Nanan Academy.

These games were built using Scratch — a block-based programming language developed at MIT — and packaged as self-contained HTML files using the TurboWarp Packager. They run directly in the Chromium browser with no internet connection required. When you play them, you are experiencing real student work: game design, programming logic, and creative problem-solving.

The cabinet can also run any HTML or JavaScript game — student projects built with p5.js, Godot (web export), or plain HTML5 work exactly the same way. See Section 4 for how students export their games, and Section 6 for how to add them.

03
CONNECTING TO WIFI

Before you can add games or ROMs remotely, the Retrocade needs to be connected to your home or office network. This is done through the RetroPie menu inside EmulationStation.

Steps

  1. Power on the cabinet and wait for EmulationStation to fully load.
  2. Press Start on the controller to open the Main Menu.
  3. Select RETROPIE and press your confirm button.
  4. Select RASPI-CONFIG from the list.
  5. Navigate to System Options → Wireless LAN.
  6. Enter your WiFi network name (SSID) and password when prompted.
  7. Select Finish and allow the system to apply settings.
📝 Note: Once connected, this machine's hostname is retrocade1 (or retrocade2 for the second cabinet). You do not need to know the IP address — the hostname works on most home and office networks.
04
EXPORTING FROM SCRATCH

Games must be exported as a single .html file before they can be added to the cabinet. This is done using the free TurboWarp Packager at turbowarp.org/packager — no software installation needed. There are two parts: packaging the game, then injecting the controller script.

Part 1 — Package the Game

  1. Open your Scratch project at scratch.mit.edu and make sure it is saved.
  2. Go to turbowarp.org/packager in your browser.
  3. Click Load project and either paste your Scratch project URL, enter your project ID, or upload your .sb3 file directly.
  4. Configure the packaging options — these are the important ones for Retrocade:
    • Start game without green flag — ✅ Enable this. On the cabinet there is no mouse to click the green flag, so the game must start automatically.
    • Full screen on start — enable this so the game fills the cabinet screen.
    • Loading screen — optional, you can customise the loading message.
  5. Click Package at the bottom of the page.
  6. When prompted, choose HTML file. A single .html file will download.
  7. Rename it clearly, e.g. space_shooter.html.
📝 Note: The exported HTML file is completely self-contained and works fully offline. This is the only game format the Retrocade system runs.

Part 2 — Inject the Controller Script

The cabinet's arcade controls work by translating joystick and button inputs into keyboard events. A small JavaScript snippet must be added to the <head> of the exported HTML file so the game receives controller input correctly.

  1. Open the exported .html file in a text editor (TextEdit on Mac in plain text mode, Notepad on Windows, or VS Code).
  2. Find the opening <head> tag near the top of the file.
  3. Paste the Retrocade controller script immediately after the <head> tag. Your teacher or the Nanan Academy team can provide the current version of this script.
  4. Save the file. The game is now ready to be added to the Retrocade folder.
📝 Note: Without the controller script, the arcade joystick and buttons will not work in the game. Every exported game needs this step before it will work on the cabinet.

Other HTML / JavaScript Games

The cabinet can run any HTML/JavaScript game — not just Scratch exports. Games built with p5.js, Godot (web export), or plain HTML5 Canvas work the same way. They still need the controller script injected into the <head> to receive arcade input.

05
ADDING ROMs

ROMs are game files for classic systems like NES, SNES, Game Boy, Arcade, and more. Retrocade supports any system that RetroPie supports. You are responsible for ensuring you own the rights to any ROM files you add.

ROMs go into the following folder on the Pi, inside a subfolder named after the system:

/home/nanan/RetroPie/roms/[system-name]/

Examples: roms/nes/  ·  roms/snes/  ·  roms/gb/  ·  roms/arcade/

📝 Note: After adding ROMs, restart EmulationStation from the Start menu so the new games appear.

Option 1: USB Stick

  1. Format a USB stick as FAT32 on your computer.
  2. Create a folder on the USB stick called retropie.
  3. Plug the USB stick into the Retrocade while it is running. Wait 30 seconds — RetroPie will automatically set up the folder structure on it.
  4. Unplug the USB and connect it to your computer. Open it — you'll find retropie/roms/ with subfolders for each system.
  5. Copy your ROM files into the correct system subfolder.
  6. Plug the USB back into the Retrocade and wait for the files to copy (watch the activity light).
  7. Unplug the USB stick, then restart EmulationStation.

Option 2: Windows Network Share (Samba)

  1. Make sure your computer and the Retrocade are on the same WiFi network.
  2. On Windows: Open File Explorer and type \\retrocade1 in the address bar and press Enter.
    On Mac: In Finder, press Cmd+K and type smb://retrocade1, then click Connect.
  3. Log in if prompted (use credentials from the Quick Reference section).
  4. Open the roms folder and drag your ROM files into the correct system subfolder.
  5. Restart EmulationStation from the Start menu.

Open a terminal on your computer and connect:

ssh nanan@retrocade1

Enter password academy when prompted. Then, from your computer (not the Pi), use scp to copy a ROM:

scp mygame.nes nanan@retrocade1:/home/nanan/RetroPie/roms/nes/

Adjust the path for the system you are copying to. After copying, restart EmulationStation from the Start menu on the cabinet.

06
ADDING STUDENT GAMES

Student games are .html files exported via the TurboWarp Packager (see Section 4). They go in the Retrocade games folder on the Pi: /home/nanan/RetroPie/roms/retrocade

USB Stick

  1. Copy the .html game file onto a USB stick.
  2. Plug the USB stick into the Retrocade.
  3. Using the file manager or network share, navigate to the Retrocade games folder and copy the file there.
  4. Restart EmulationStation.

Network Share (Samba)

  1. Connect to the cabinet via network share (see Section 5 for steps).
  2. Navigate to the Retrocade games folder and drag your .html file in.
  3. Restart EmulationStation.

From your computer's terminal, copy the file directly with scp:

scp mygame.html nanan@retrocade1:/home/nanan/RetroPie/roms/retrocade/
📝 Note: EmulationStation can be restarted from the Start menu on the cabinet — you don't need to do it over SSH.
